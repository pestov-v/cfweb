# Інструкція з налаштування GitHub Pages для CFWeb Gallery

## Проблема

Поточна структура проекту:
```
cfweb/
├── cfweb.ukrpay.net_*.webp (зображення в корені)
├── gallery/
│   └── index.html (посилається на ../cfweb.ukrpay.net_*.webp)
└── README.md
```

При публікації з папки `gallery` на GitHub Pages, шляхи `../` не працюють правильно, оскільки зображення знаходяться поза папкою публікації.

## Рішення (Варіант 1 - Рекомендований)

Перемістити всі зображення в папку `gallery/images/` і оновити шляхи в HTML.

### Крок 1: Створити папку для зображень

```bash
mkdir -p gallery/images
```

### Крок 2: Перемістити зображення

```bash
mv cfweb.ukrpay.net*.webp gallery/images/
```

### Крок 3: Оновити шляхи в HTML

Замінити всі `../cfweb.ukrpay.net_*.webp` на `images/cfweb.ukrpay.net_*.webp`

### Крок 4: Налаштувати GitHub Pages

1. Перейдіть в репозиторій на GitHub
2. Settings → Pages
3. У розділі "Build and deployment":
   - Source: виберіть "Deploy from a branch"
   - Branch: виберіть `main` (або вашу основну гілку)
   - Folder: виберіть `/gallery`
4. Натисніть "Save"

### Крок 5: Перевірити результат

Після деплою (зазвичай 1-2 хвилини), ваш сайт буде доступний за адресою:
```
https://your-username.github.io/repository-name/
```

---

## Рішення (Варіант 2 - Альтернативний)

Залишити структуру як є, але публікувати з кореня репозиторію.

### Крок 1: Перемістити index.html в корінь

```bash
mv gallery/index.html index.html
```

### Крок 2: Оновити шляхи в HTML

Замінити всі `../cfweb.ukrpay.net_*.webp` на `cfweb.ukrpay.net_*.webp` (прибрати `../`)

### Крок 3: Налаштувати GitHub Pages

1. Settings → Pages
2. Source: "Deploy from a branch"
3. Branch: `main`
4. Folder: `/ (root)`
5. Save

### Результат

Сайт буде доступний за адресою:
```
https://your-username.github.io/repository-name/
```

Але URL буде: `https://your-username.github.io/repository-name/` (без `/gallery`)

## Рекомендації

**Варіант 1** краще, тому що:
- ✅ Всі файли сайту в одній папці (`gallery/`)
- ✅ Легше керувати структурою
- ✅ Можна додавати інші файли в корінь без впливу на сайт
- ✅ Чистіша структура проекту

## Перевірка після налаштування

1. Перевірте, що всі зображення завантажуються
2. Перевірте, що модальні вікна відкриваються з правильними зображеннями
3. Перевірте на мобільних пристроях (responsive design)

---

## Troubleshooting

### Зображення не завантажуються

- Перевірте шляхи в HTML (мають бути відносні до `gallery/`)
- Перевірте, що всі файли `.webp` знаходяться в правильній папці
- Перевірте консоль браузера (F12) на наявність помилок 404

### GitHub Pages не оновлюється

- Перевірте, що зміни запушені в правильну гілку
- Зачекайте 1-2 хвилини (деплой не миттєвий)
- Перевірте Actions tab на наявність помилок білду

### 404 помилка

- Перевірте налаштування Source/Folder в Settings → Pages
- Переконайтеся, що `index.html` знаходиться в папці, вказаній в налаштуваннях
